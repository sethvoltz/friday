# Mobile UX

Friday is mobile-first, not mobile-responsive-as-an-afterthought. Every page is fully usable on a phone, including chat with attachments, agent inspection, and dashboard tables.

## Layout primitives

- **Fixed header** on all viewports. No bottom tab bar.
- **The document is the only scroller — chat included.** The chat transcript is an inert in-flow element that scrolls the document/window like every other page; there is no nested or `position: fixed` chat scroller. The pinned bars (header, sidebar, composer) are `position: fixed` overlays layered over the document scroll, not competing scrollers. A fixed full-viewport overlay scroller intermittently lost the touch-routing fight to the document on mobile WebKit, dragging the page as a chunk — see ADR-039; don't reintroduce one.
- **Header nav — one unified priority+ mechanism at every width.** Links live in `.header-right` and overflow **one at a time** into a `More` menu as the header narrows; on a phone every link ends up in `More` (which reads as the hamburger), so there is no separate breakpoint that dumps the whole nav at once. The `More` button only mounts when the overflow bucket is non-empty. Sidebar collapses behind a sheet/drawer triggered from the chat header. Friday 👑 always at the top of the drawer.
  - **Bounce-free by construction.** An off-screen ghost copy of the links supplies intrinsic widths; the available-width signal is `.header-right`'s box width — the flex _leftover_ after the content-stable `header-left`, which does **not** change when a link moves between the visible row and `More`. That severs the classic priority+ feedback loop (hide a link → container widens → link reappears → oscillate), so the visible count is a pure, monotonic function of the measured width. See `services/dashboard/src/routes/+layout.svelte`.

## List pages

- Card layout on mobile, table/grid on desktop.
- Sentinel-paginated via `IntersectionObserver` — top + bottom sentinels load older / newer pages.
- Avoid full-array renders for anything that could grow unbounded (chat, sessions, mail, evolve proposals, logs).

## Forms

- One field per row.
- Inputs ≥ 44px tall (touch target minimum).
- Full-width buttons.
- Bottom-anchored CTA (primary action stays in thumb reach).
- Modals become full-screen sheets on mobile.

## Detail pages

- Full-screen on mobile, in-place pane on desktop. Same content, different framing.

## Touch targets

- All interactive targets ≥ 44×44 px. Includes sidebar agent rows, autocomplete entries, attachment thumbnails.

## PWA

- **Manifest + icons only.** No service worker; offline mode is meaningless when the daemon is unreachable.
- "Add to Home Screen" gives the app-like launch with a custom icon.
- Manifest lives at `services/dashboard/static/manifest.webmanifest`.
- Icons: `icon-192.png`, `icon-512.png`, `icon-maskable-512.png` under `static/`. Placeholder generated by `scripts/generate-icons.mjs`; run with `--source=path/to/logo.svg` to regenerate from a real logo.

## Chat virtualization

- Hand-rolled in ~150 LOC using `IntersectionObserver` sentinels at top/bottom + DOM windowing. No React-flavored libraries; Svelte 5 runes + ResizeObserver throughout.
- Window the rendered range — keep ~200 messages in DOM; recycle as the user scrolls.
- Auto-pin-to-bottom only when the user is already at the bottom (track `isAtBottom`).
- Bottom-sentinel: when off-screen, show a "↓ new messages" button.

(Implementation pending — see `docs/roadmap.md`.)

## Mobile autocomplete (the slash-command picker)

- Reference: Slack mobile.
- Tap-to-insert must preserve keyboard focus. Standard `click` blurs the input on mobile, which then dismisses the picker.
- Use `pointerdown` + `preventDefault` on the autocomplete entry. The keyboard stays up; the slash-completion lands in the input; the user keeps typing.

## Camera and gallery uploads

- `<input type="file" accept="image/*" capture="environment">` for "open camera" affordance.
- Standard `<input type="file" multiple>` for gallery picker.
- HEIC files are converted to PNG by the upload handler (ADR-007 + `attachments.ts`); the dashboard doesn't care about the source format.

## iOS soft keyboard — visualViewport pinning

iOS raises the soft keyboard asynchronously after a text field is focused. `position: fixed` elements use the **layout viewport** as their reference, which does not shrink when the keyboard opens — so a `bottom: 0` composer bar stays pinned to the bottom of the layout viewport and ends up hidden behind the keyboard.

Fix implemented in `b32595b`:

- **`--vv-offset-bottom`** (set in `+layout.svelte` `vvUpdate`): `max(0, window.innerHeight - vv.height)`. This is the true keyboard height in layout-viewport coordinates — stable once the keyboard is up because `vv.height` only changes when the keyboard appears/disappears, not during scroll. Set to `0` when no text field is focused so scroll animations don't reposition the composer.
- **`.chat-input-floating`** in `ChatShell.svelte` applies `transform: translateY(calc(-1 * var(--vv-offset-bottom, 0px)))` to lift the composer above the keyboard. `transform` works in the layout-viewport coordinate system, so this precisely counteracts the overlap.
- **`scrollIntoView` on focus** (`ChatInput.svelte`): fires after a 100ms delay so the keyboard has begun raising before we scroll. Uses `behavior: "instant"` — `behavior: "smooth"` is unreliable on WebKit (bug #238497).

`vvUpdate` is wired to `visualViewport` `resize` events plus `focusin`/`focusout` on the document. The initial keyboard-open scroll is captured when `resize` fires (iOS shrinks `vv.height`, which fires `resize`, at which point `vv.offsetTop` already reflects the new position). Subsequent user scrolling does not move the header or composer.

## Public reachability + Cloudflare Tunnel

- The dashboard is the only thing exposed publicly. CFT terminates at `127.0.0.1:5173`.
- BetterAuth gates everything behind login. No public sign-up endpoint (ADR-008).
- The daemon never sees the public internet — only the dashboard, gated by BetterAuth.
- The 20s `:keepalive` heartbeat on the SSE stream prevents idle disconnect through CFT.
