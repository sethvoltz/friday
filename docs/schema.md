# Database schema

Single SQLite file at `~/.friday/db.sqlite`, WAL mode. Schema defined in `packages/shared/src/db/schema.ts` via Drizzle ORM. Migrations are generated by `pnpm --filter @friday/shared drizzle:generate` and applied at daemon startup.

This doc is a reference index for the tables and their relationships. The TypeScript schema in `schema.ts` is the source of truth.

## Tables

### Auth (BetterAuth-managed)

`accounts`, `sessions`, `users`, `verification` — declared in `schema.ts` for typed access from the rest of the app, but BetterAuth's CLI owns the migrations (ADR-015). Field names match BetterAuth's defaults.

### `turns`

Mirror of Claude session JSONL files, plus dashboard-write rows (chat replies, etc.).

| Column | Notes |
|---|---|
| `id` | autoincrement |
| `session_id` | Claude SDK session id |
| `agent_name` | nullable; identifies the speaker for synthesized rows |
| `turn_index` | byte offset into the source JSONL (ADR-012). Monotonic + unique within session. |
| `ts` | unix ms |
| `role` | `user` / `assistant` / `system` |
| `kind` | `user` / `assistant` / `system` / `tool_use` / `tool_result` |
| `content_json` | raw JSONL line |
| `source_file` | absolute path to the JSONL |
| `source_byte_off` | duplicate of `turn_index`; kept for potential schema migrations |
| `last_event_seq` | EventBus sequence at the moment this row was committed (ADR-004) |

UNIQUE `(session_id, turn_index)`.

### `turns_fts` (FTS5)

Contentless FTS index over `turns.content_json`.

### `mail`

| Column | Notes |
|---|---|
| `id` | autoincrement |
| `from_agent`, `to_agent` | agent names |
| `type` | `message` / `notification` / `task` |
| `delivery` | `pending` / `delivered` / `read` / `closed` |
| `body` | markdown |
| `meta_json` | optional structured metadata |
| `ts`, `read_at`, `closed_at` | unix ms |

### `tickets`

| Column | Notes |
|---|---|
| `id` | TEXT primary key, format `FRI-1234` |
| `title`, `body`, `status`, `kind`, `assignee` | self-explanatory |
| `meta_json`, `created_at`, `updated_at` | |

`status` ∈ `open` / `in_progress` / `done` / `blocked` / `closed`.
`kind` ∈ `task` / `epic` / `bug` / `chore`.

### `ticket_relations`, `ticket_comments`, `ticket_external_links`

- Relations: `(parent_id, child_id, kind)`.
- Comments: `(id, ticket_id, author, body, ts)`.
- External links: `(ticket_id, system, external_id, url, meta_json, linked_at)` — composite PK; index on `(system, external_id)`. ADR-006: tickets are system-agnostic; integrations slot in here.

### `attachments`

| Column | Notes |
|---|---|
| `sha256` | TEXT primary key (ADR-007) |
| `filename`, `mime`, `size_bytes` | |
| `uploaded_at` | unix ms |
| `first_turn_id` | nullable; first turn that referenced this attachment |

### `agents`

The agent registry — replaces the old `agents.json` (ADR-013).

| Column | Notes |
|---|---|
| `name` | TEXT primary key |
| `type` | `orchestrator` / `builder` / `helper` / `scheduled` / `bare` |
| `status` | `idle` / `working` / `stalled` / `error` / `killed` |
| `session_id` | current SDK session, nullable |
| `parent_name` | nullable; orchestrator name for builder/helper/bare |
| `worktree_path`, `branch`, `ticket_id` | builder-specific |
| `meta_json` | reserved for future per-type fields |
| `created_at`, `updated_at` | |

INDEX on `type`, INDEX on `(status, updated_at)`.

### `schedules`

| Column | Notes |
|---|---|
| `name` | TEXT primary key |
| `cron`, `run_at` | one or the other; cron for recurring, run_at for one-shot |
| `task_prompt` | the first-turn prompt for the spawned scheduled agent |
| `paused` | bool, default false |
| `next_run_at`, `last_run_at`, `last_run_id` | unix ms / synthetic run id |
| `meta_json`, `created_at`, `updated_at` | |

INDEX on `next_run_at`.

### `memory_entries`

Mirror of file-based memory at `~/.friday/memory/entries/<id>.md`. The file is the source of truth; this row is the searchable index.

| Column | Notes |
|---|---|
| `id` | slug |
| `title`, `content`, `tags_json`, `created_by` | |
| `created_at`, `updated_at` | ISO strings |
| `file_mtime` | unix ms; staleness check |
| `recall_count`, `last_recalled_at` | bumped by `touchRecall(id)` on every `GET /api/memory/:id` |

`memory_fts` (FTS5) indexes title + content + tags for hybrid keyword search with recall-frequency boost.

### `db_meta`

Key/value metadata. Currently used for: `ticket_counter` (monotonic counter for `FRI-N` ids).

## Conventions

- All timestamps are unix milliseconds (`integer`) unless documented as ISO strings.
- All JSON columns use `_json` suffix.
- `nullable` columns are explicit in the Drizzle schema; absence means `NOT NULL`.
- All indexed columns are noted above.

## Migrations

```bash
# After editing schema.ts
pnpm --filter @friday/shared drizzle:generate

# Generated SQL lands in packages/shared/src/db/drizzle/
# The daemon applies pending migrations on startup via runMigrations().
```

BetterAuth tables are created by `@better-auth/cli` separately — see ADR-015.
